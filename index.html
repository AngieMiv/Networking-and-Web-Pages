Accessing the Web

Accediendo a la Web

When someone enters a website URL, several steps happen behind the scenes:

Cuando alguien escribe la dirección de una web, ocurren varios pasos en segundo plano:

  1. DNS translates the domain name to an IP address.
  2. TCP/IP establishes a connection with the server.
  3. HTTP requests the HTML/CSS/JS content from the server.
  4. Web server (Apache/Nginx) sends the requested files.
  5. Linux (Debian/Ubuntu) runs the server and hosts the files.
  1. DNS traduce el nombre de dominio a una dirección IP.
  2. TCP/IP establece una conexión con el servidor.
  3. HTTP solicita el contenido HTML/CSS/JS al servidor.
  4. Servidor web (Apache/Nginx) envía los archivos solicitados.
  5. Linux (Debian/Ubuntu) ejecuta el servidor y aloja los archivos.
